# 사용자 요구 기반 제품 방향 정리 (2026-03-01)

아래는 지금까지 요청을 통합한 **실제 선호 방향**입니다. 이후 구현/우선순위 판단 기준으로 사용합니다.

## 1) 전체 방향
- “깔끔”만이 아니라, 연습 의욕이 생기는 디자인/피드백/성취감이 중요
- 웹 초보도 계속 쓸 수 있게 UI는 직관적이어야 함(검색/필터/토글/기본값)
- 기능은 많아져도 화면은 복잡하지 않게, 핵심은 항상 전면에

## 2) 대시보드
- 드래그 없이 한 화면에 핵심 정보가 보이는 레이아웃 우선
- 화면 폭이 줄어도 HUD/타이머/핵심 카드가 깨지지 않도록 반응형 필수
- 핵심 카드:
  - 유저 HUD(레벨/총XP/다음 레벨까지/총 연습시간/배지)
  - 세션 타이머
  - 이번 주/월 진행
  - TODO
  - 곡 바로가기(앨범 커버 중심, 즉시 시작)
  - 대시보드 전용 이미지 영역(기록장 연동보다 직접 관리 선호)
- 불필요한 문구는 줄이고 정보 밀도는 높게

## 3) 세션/XP/레벨 밸런스
- 50 레벨 만렙, 장기 성장형
- “2년 동안 주 3회 30분 연습” 수준에서 만렙 근접/달성되는 밸런스 선호
- 세션 XP, 업적 XP, 퀘스트 XP 간 상대 밸런스 재조정 필요
- 세션 삭제/수정 시 XP와 레벨이 정확히 역반영되어야 함
- 일일 연습 XP 상한 등 악용 방지 정책 필요
- 테스트용 관리자 기능:
  - 다음 레벨까지 필요 XP 즉시 지급
  - 초기화(완전 리셋)

## 4) 유저 HUD/배지/보상감
- 배지는 티어별(브론즈/실버/골드/플래티넘/다이아/챌린저) + 레벨 구간별 점진 화려화
- 10단위 티어 전환 시 “새 티어 시작”이 명확히 보이게(다시 단순해짐)
- 레벨업/업적 달성 시 탭과 무관하게 알림/이펙트 표시
- 칭호는 모든 레벨에서 차등 제공(해금 항목과 분리)

## 5) 연습 스튜디오
- 먼저 모드 선택(곡 연습 / 드릴 연습) 후 관련 선택 UI 표시
- 곡/드릴 검색 필수, 즐겨찾기/최근 사용 우선 노출
- 곡 연습:
  - YouTube iframe은 크게, 링크 입력 UI가 화면을 먹지 않게
  - 앨범 커버가 있으면 분위기 배경/참고 영역에 활용
- 드릴 연습:
  - 이미지 중심(확대/전체화면), 배킹트랙은 선택적으로 사용
  - 드릴별 기본 배킹트랙 지정 가능 + 장르/BPM 필터 선택

## 6) 라이브러리(곡/드릴/배킹)
- 곡/드릴 추가 폼은 기본 펼침이 아니라 버튼/토글로 열기
- 상태 전환은 단일 버튼 순환이 아니라 명시적 선택
- 곡 상태 체계(요청 반영):
  - 시작 전: 목표/예정/카피중
  - 진행 중: 시작/루프연습/연습중
  - 완료: 마무리/공연완료/포기
- 즐겨찾기 지원(상단 우선/즐겨찾기 필터)
- 장르/태그는 표준화 + 그룹형 다중선택
- 추천곡과 곡 라이브러리 장르 풀은 동일하게 유지
- 추천곡 데이터는 `song_ladderv2.csv` 기준 활용(장르+분위기)

## 7) 기록 탭(세션 기록/돌아보기/기록장)
- 세션 기록:
  - 수정/삭제가 정확히 동작
  - 노트 중심으로 가독성 개선
  - 곡/드릴 표시는 존재하는 값만 노출(불필요한 `-` 최소화)
- 돌아보기:
  - 단순 로그보다 성취 시각화 중심
  - 주/월/연/사용자 기간 필터 통합 UX
  - 곡/드릴/장르/목적/상태 기반 집계
- 기록장(구 갤러리/미디어 통합):
  - 게시글(제목/본문) + 다중 첨부(사진/영상/오디오/텍스트)
  - 곡/드릴 다중 연결, 종류 태그, 검색/필터
  - 리스트/갤러리 뷰 전환
  - 본문 입력 크게, 첨부를 글 흐름 안에서 다루는 UX 선호

## 8) 업적/퀘스트
- 업적 수량 축소(대략 30~50개), 의미 없는 항목 정리
- 전면 한글화(이름/설명/태그), 영어 내부코드 노출 최소화
- 설명은 간결 표시 + 상세는 툴팁/정보 버튼
- 티어 색상/아이콘/유형 구분을 명확히
- 히든 업적 일부 유지, 힌트는 제공
- TODO와 수동 퀘스트 연동:
  - TODO 생성 시 수동 퀘스트 자동 생성
  - 완료 시 TODO 자동 정리
  - 실패/재개/삭제 액션은 작은 메뉴(우클릭/점 3개) 선호

## 9) 연습 도구
- 메트로놈:
  - 박자 로직 정확성(예: 4/4면 4카운트)
  - UI 가림 최소화(위치 고정 또는 오버레이 정책 명확화)
  - BPM 입력 UX 개선(클리핑보다 유효성 검증 방식)
  - 시각화 제공(강박 강조)
- 탭 메이커:
  - 빠른 입력(클릭+키보드 단축)
  - 길이/점음표/셋잇단 선택형
  - 반복 입력 편의/실행취소
  - JSON 저장/불러오기, 이미지 내보내기, 재생 기능

## 10) 빌드/배포
- 사용자는 `build_exe.ps1` 실행으로 결과물 확인하는 흐름
- 빌드 후 실제 변경 반영 여부를 눈에 띄게 검증 가능해야 함
- exe 아이콘은 `designPack/icon.png` 활용 요구 있음

## 11) 즉시 참고할 우선순위
1. 레이아웃 안정화(드래그 없이 핵심 UI 안 깨짐)
2. XP/레벨 밸런스 재정의 + 역반영 정확성
3. 연습 스튜디오 곡/드릴 선택 UX 단순화 + iframe/이미지 품질
4. 라이브러리/기록장 검색·필터·그룹화 고도화
5. 업적/태그 한글화 및 데이터 정리(원본 CSV 포함)
