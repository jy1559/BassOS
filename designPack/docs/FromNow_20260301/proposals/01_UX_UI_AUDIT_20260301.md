# UX/UI 점검 메모 (2026-03-01)

이 문서는 **코드 구조 점검 + 대화/스크린샷 피드백 기반**으로 작성했습니다.
(현재 환경에서 GUI 실기동 관찰은 제한적이므로, 정적 점검 결과와 사용자 피드백을 함께 반영)

## A. 즉시 개선 우선 (체감 큰 항목)
1. 대시보드 overflow/잘림
- 증상: HUD/타이머 카드가 작은 창에서 줄바꿈/잘림
- 원인 후보: 고정 그리드(3x4) + 카드 최소 폭/폰트 크기 고정
- 개선
  - 핵심 카드(HUD/타이머)는 `minmax()` 기반으로 최소폭 보장
  - 좁은 폭에서는 자동 1열 또는 2열 재배치
  - HUD 수치 폰트에 clamp 적용

2. 연습 스튜디오 가시성
- 증상: 참고 링크/iframe/패널 비중이 상황에 따라 산만
- 개선
  - 곡 모드: 앨범 커버 작게, iframe 우선(큰 비율), 링크 셀렉트는 상단 1줄
  - 드릴 모드: 이미지 중심 + 배킹은 우측 보조 패널. 이미지 없을 경우 배킹 크게
  - 시작 패널은 대상 선택 후 기본 접힘 유지

3. 기록장 작성 UI 밀도
- 증상: 태그/연결 선택 영역이 길고 난잡
- 개선
  - 탭형 분리(본문/연결/첨부)
  - 본문 편집 영역 고정 높이 확대
  - 연결 곡/드릴 목록은 기본 접힘 + 즐겨찾기 우선

## B. 구조 개선 우선
1. 용어 정규화
- 목표: Song/Drill/Quest/Achievement 전반에서 한글/영문 혼재 최소화
- 작업
  - 태그/상태/카테고리 표시용 전용 dictionary 단일화
  - 내부 코드 키와 UI 라벨 분리

2. 설정-밸런스 연결 가시화
- Critical 값 변경 시 예상 레벨링 속도 preview 제공
- 예: “주 3회 30분 기준 Lv50 예상 주차” 계산 카드

3. 레거시 API 정리
- records 중심 통합 완료 후 gallery/media UI 잔여 흐름 단계적 정리

## C. 시각 완성도 개선
1. 배지 시스템
- 10레벨 단위 티어 시작 시 단순화 + 후반 화려화 패턴 명확화
- 배지 미리보기(현재/다음)

2. 테마 품질
- dark/neon/midnight 텍스트 대비 체크리스트화
- 카드/표/툴팁/입력창 대비값 자동 점검

3. 모션 피드백
- 레벨업/업적 달성 시 글로벌 오버레이를 탭 불문 확실히 표시
- 과도한 애니메이션 방지를 위해 intensity 단계별 프리셋

## D. 데이터 관점 개선
1. song_ladderv2 정규화
- 장르/분위기/테크닉 태그를 앱 표준과 매핑 테이블로 연결
- 추천곡 ↔ 곡 라이브러리 장르 풀 동기화 자동화

2. 업적 슬림화
- 현재 46개에서 중복/의미 약한 항목 정리
- 30~50개 유지 + 설명 가독성 강화 + 히든 소수 유지
