# Bass Practice Game (로컬 웹앱) 시스템 설계 v1

이 문서는 “베이스 연습을 **끊기지 않게** 만들기 위한 게이미피케이션 설계”입니다.  
구현은 **로컬 CSV를 데이터베이스로** 사용하고, 웹 UI에서 **세션 시작/종료(타이머)** → 자동 XP 적립 → 업적/해금/보스 → 기록(오디오/영상)까지 연결합니다.

---

## 1) 디자인 목표(문제 정의)

### 반드시 해결해야 하는 것
- **주 3회 이상 베이스를 잡는 습관**을 만든다.
- 합주/공연 급한 때만 몰아서 하지 않고, **기본기(Core)**가 꾸준히 누적되게 한다.
- “기록이 귀찮아서 시스템이 망하는” 문제를 최소화한다.
- XP/레벨이 단순 숫자 놀이가 되지 않게, **해금/트로피/영상 결과물**로 남긴다.

### 기본 철학
- **시간 XP는 ‘시작 보상’**이고, 진짜 큰 보상은  
  **완주(보스) / 녹음(증거) / 품질(메트로놈/클린) / 다양성(장르/기술 컬렉션)**에 준다.
- 벌점/죄책감 대신, **백필(늦게 기록)**과 **5분 세션**도 인정해서 “복귀 장벽”을 낮춘다.

---

## 2) 핵심 데이터 원칙: XP 장부(Events)가 단일 진실 소스

- 모든 XP는 `data/events.csv` 한 파일에 기록된다.
- 세션(연습)도, 업적 수령도, 퀘스트 완료도 전부 “이벤트 1줄”이다.
- 스탯(총 XP, 레벨, 랭크, 진행바)은 events.csv의 합계/집계로만 계산한다.

장점:
- 규칙이 바뀌어도 과거 데이터는 유지된다.
- “XP를 어디서 얻었는지”가 투명해서 동기부여가 유지된다.

---

## 3) XP 설계(밸런스 핵심)

### 3-1) 세션 XP(타이머 기반, 자동 계산)
- 기본 XP(세션 시작+시간):
  - `base_xp = min(max_base_xp, start_bonus + per_10min * floor(duration_min/10))`
  - 설정: `start_bonus=15`, `per_10min=12`, `max_base_xp=80` (settings.json)

예:
- 10분: 15 + 12*1 = 27 XP
- 30분: 15 + 12*3 = 51 XP
- 60분: 80 XP(캡)

### 3-2) 품질/의도 보너스(체크박스)
세션 종료 시 체크:
- Core 워밍업(+10)
- 메트로놈 2&4(+10)
- 1마디 1번 클릭(+20)
- 녹음(오디오 +15 / 영상 +25)
- 귀카피 세션(+10)
- 이론(+8)
- 슬랩(+10)
- 클린/뮤트(+10)
- A/B 비교(+15)

**권장 UX:** 체크박스 + (선택) 증거 링크/파일.  
(녹음 보너스는 링크/파일이 있으면 더 “진짜 보상” 느낌이 남)

### 3-3) 보스/퀘스트 XP
- 주간 퀘스트(Weekly): 주3회 + Core2회 달성 시 보상 XP(기본 150 + 120)
- 월간 보스(Monthly Boss): 곡 풀테이크(300) 또는 귀카피 보스(320)

> 합주/공연은 “세션 XP + 작은 보너스(10~40)” 정도로만.  
> 대신 업적/트로피(날짜·장소 기록)로 의미를 크게 남긴다.

## 4) 레벨 커브(초반 자주, 후반 꾸준히)

레벨업 필요 XP(다음 레벨까지) = **완만한 가속(2차식)**

- `xp_to_next(L) = a + b*(L-1) + c*(L-1)^2`
- 기본값: a=200, b=30, c=2

특징:
- 초반(1~10): 자주 레벨업(동기 점화)
- 중반(10~20): 주 3회 페이스로 2~3주에 1레벨 수준
- 후반(20+): 꾸준히 해야 올라감(장기 목표)

랭크(예시):
- 브론즈(Lv1~7) → 실버(Lv8~14) → 골드(Lv15~22) → 플래티넘(Lv23~31) → 다이아(Lv32+)

### 레벨업 예상 페이스(예시)

- 가정: **주 3회**, 회당 30~45분 + 체크박스 보너스 일부(메트로놈/코어/녹음 중 1~2개)

- 평균 XP: **주 450~650 XP** 정도가 자연스럽게 나오는 밸런스


| 목표 레벨 | 필요 누적 XP(대략) | 주 500XP 기준 예상 | 체감 |
|---:|---:|---:|---|

| Lv2 | 200 | 0.4주 | 초반 점화 |
| Lv5 | 1,008 | 2.0주 | 초반 점화 |
| Lv10 | 3,288 | 6.6주 | 습관 고정 |
| Lv15 | 7,168 | 14.3주 | 습관 고정 |
| Lv20 | 13,148 | 26.3주 | 6개월 목표 |
| Lv25 | 21,728 | 43.5주 | 장기 |
| Lv30 | 33,408 | 66.8주 | 장기 |

**튜닝 팁**
- 레벨업이 너무 느리면: `per_10min`을 12→13~15로, 또는 (a,b,c) 중 **a를 200→170**으로 낮추기
- 레벨업이 너무 빠르면: `max_base_xp`를 80→70으로, 또는 **c를 2→3**으로 올려 후반을 더 묵직하게
- “기본기 유도”를 더 강하게 하고 싶으면: Core 보너스(+10) 또는 주간 Core 퀘스트 XP를 소폭 상향

## 5) 레벨업 ‘해금’ 설계(숫자 말고 게임 느낌)

`data/unlockables.csv`로 해금 목록을 관리한다.

### 해금 아이디어(권장)
- **Theme 스킨**: 다크/네온펑크/재즈라운지 등
- **Feature**:
  - 업적 토스트/애니메이션
  - 보스 카드 갤러리(완주 곡 카드 수집)
  - 공유 카드(이미지로 저장)
  - A/B 비교 플레이어(원곡 vs 내 테이크)
  - 스킬 트리 뷰(4개 분야 진행 시각화)
- **Token**(선택):
  - 퀘스트 리롤권(주1회), 스트릭 세이브 등

> “레벨이 오를수록 UI가 화려해지고, 할 수 있는 기능이 늘어난다”가 핵심.
지금 개수가 그렇게 많지 않아서, 좀 레벨 올리는거에 대해 흥미를 유발할 수 있도록 다양한 해금 아이디어 추가하면 좋을 것 같음
근데 필수적인걸 막아놓으면 오히려 반감생기니 캐릭터 꾸미기, 테마 등 미적 요소면 좋을 것 같고, 유용한거? 잇으면 좋겠음. 역시 대부분은 성취감용입면 좋을듯
베이스 연습이랑 관련 없어도 뭔가 재미 느낄 수 있거나 동기 유발하면 좋음

## 6) 업적 시스템(매우 중요)

### 6-1) 구조
- `data/achievements_master.csv` : 업적 정의(티어 포함)
- 업적 “수령”은 events.csv에 `event_type=ACHIEVEMENT_CLAIM` 이벤트로 기록

### 6-2) 티어
- 대부분 **동/은/금/백금/다이아**
- 단발(기념 업적)은 tier_name=단발
- 단발도 예상 난이도에 따라 티어 구분해도 좋을듯
- 재미용 단발 업적도 추가하면 좋고, 내가 못보는 히든 업적 숨겨놔도 재밌을 것 같음. 힌트정도 주고

### 6-3) 룰 엔진(구현 난이도 낮춘 버전)
업적 rule_type(지원 타입) 최소 세트:
- `count_events`: 조건에 맞는 이벤트 개수
- `sum_duration`: 조건에 맞는 세션 누적 분
- `sum_xp`: 누적 XP(전체)
- `distinct_count`: 특정 필드의 서로 다른 값 개수(예: song_library_id)
- `streak_weekly`: 주간 연속 달성
- `streak_monthly`: 월간 연속 달성
- `boss_monthly`: “월간 보스 달성한 달” 누적
- `manual`: 버튼으로 수동 수령(날짜 기록형/커뮤니티/공연 장소 등)

> 자동 업적 + 수동 업적을 섞으면 “재밌는 업적”을 크게 늘릴 수 있다.

## 7) 곡/연습자료 구조(두 개로 분리)

### 7-1) Song Ladder (`data/song_ladder.csv`)
- 추천/백로그용 “카탈로그”
- 수백 곡을 넣어두고 필터링(난이도/에너지/장르)

### 7-2) Song Library (`data/song_library.csv`)
- 내가 실제로 연습하는 곡(카피/악보/합주/공연/그냥 재밌어서)
- 상태(Status): 자주 듣는 노래 / 할까 고민중 -> 카피중(이미 악보 있는 경우 이 단계 건너뜀) -> 준비 -> 연습중 -> 연습완료 -> 마스터 / 공연 완료 / 중단 등등...
- 목적(Purpose): 카피 연습 / 공연 / 실력향상 / 재미
- 위 예시 기본으로, 입/출력 한글로 이해하기 좋게 다양하게 정리

세션 기록에서 song_library_id를 선택하면:
- “이 곡을 얼마나 했는지” 자동으로 누적되고
- 보스(풀테이크) 완료 시 보스 카드가 생긴다.

## 8) 기록 귀찮음(UX) 해결 장치

### 필수 UX
- **Start Session / Stop Session** 버튼 2개(타이머 자동)
- Stop 시 “체크박스+드롭다운+파일/링크” 최소 입력
- **Quick Log(10분)** 버튼: 타이머 없이 15초 입력으로 기록
- **백필(나중 기록)**: 가능하되 XP는 0.5배(설정)로 “부담 없이 복귀”

### 권장 UX
- 실수/미완벽 허용: “어글리 테이크도 인정”
- 녹음/영상은 사다리로 유도:
  - 오디오 15~30초 → 영상 30~60초(언리스트 OK) → 풀테이크 → 공개 업로드

## 9) 미디어(유튜브/MP3) 재생 & 비교

- Song Library에 원곡 링크(YouTube) 저장
- 내 테이크는
  - 로컬 mp3/영상 파일을 `media/`에 저장하거나
  - URL로 저장(구글드라이브/유튜브 비공개 등)

**A/B 플레이어(해금 기능)**:
- 왼쪽: 원곡(YT iframe)
- 오른쪽: 내 테이크(audio/video)
- 같은 구간 반복/스크럽(가능하면)

## 10) 포함된 데이터 파일(요약)

- `data/achievements_master.csv` : 업적 정의(500+)

- `data/song_ladder.csv` : 추천 곡 샘플
